# 设置

AppPorts 的设置页面可通过主窗口右上角的齿轮图标访问。

## App Store 与 iOS 设置

| 设置项 | 说明 | 默认值 |
|--------|------|--------|
| App Store 应用迁移 | 允许迁移 App Store 应用。macOS 15.1 以下需手动开启 | 关闭 |
| iOS 应用迁移 | 允许迁移 iOS/iPadOS 应用（Mac 版） | 关闭 |

::: tip 💡 macOS 15.1+ 用户
macOS 15.1 及以上版本支持 App Store 应用原生安装到外置硬盘。建议在 AppStore 设置中开启「下载并安装大型 App 到独立存储盘」，而非使用 AppPorts 的迁移开关。
:::

## 签名设置

| 设置项 | 说明 | 默认值 |
|--------|------|--------|
| 自动重签名 | 数据目录迁移后自动对关联应用执行 Ad-hoc 重签名 | 关闭 |
| 开机自动重签名 | 每次用户登录时自动对签名已失效的已迁移应用执行 Ad-hoc 重签名 | 开启 |

开启「自动重签名」后，每次数据目录迁移完成后会自动备份原始签名并执行重签名，避免迁移后出现「已损坏」提示。

开启「开机自动重签名」后，系统会安装一个 LaunchAgent（`com.shimoko.AppPorts.re-sign`），在每次用户登录时扫描签名备份记录，自动对 ad-hoc 签名已失效的应用重新签名。重签名日志写入 AppPorts 默认日志文件。

::: tip 💡 已链接应用的自动重签名
对于已链接应用（状态为「已链接」），自动重签名会自动解析 Stub Portal 壳或符号链接背后的**外部真实应用路径**，确保签名变更作用于实际的应用包，而非无效的本地壳。备份和重签名操作均以真实应用的 Bundle ID 为标识。
:::

## 日志设置

| 设置项 | 说明 | 默认值 |
|--------|------|--------|
| 启用日志记录 | 将运行日志写入文件 | 开启 |
| 最大日志大小 | 日志文件超出后自动截断较旧的一半 | 2 MB |
| 日志位置 | 日志文件保存路径 | `~/Library/Application Support/AppPorts/AppPorts_Log.txt` |

### 日志操作

| 操作 | 说明 |
|------|------|
| 在 Finder 中查看 | 打开日志文件所在目录 |
| 导出诊断包 | 生成包含日志、操作记录、系统信息的 ZIP 文件 |
| 清空日志 | 清除当前日志文件内容 |

日志详细说明请参阅[日志与诊断](/logging)。
