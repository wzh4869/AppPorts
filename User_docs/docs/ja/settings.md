# 設定

AppPorts の設定ページは、メインウィンドウの右上隅にある歯車アイコンからアクセスできます。

## App Store & iOS 設定

| 設定 | 説明 | デフォルト |
|------|------|----------|
| App Store アプリ移行 | App Store アプリの移行を許可。macOS 15.1 未満では手動で有効にする必要があります | オフ |
| iOS アプリ移行 | iOS/iPadOS アプリ（Mac 版）の移行を許可 | オフ |

::: tip 💡 macOS 15.1+ ユーザー
macOS 15.1 以降は、App Store アプリの外付けドライブへのネイティブインストールをサポートしています。AppPorts の移行トグルを使用する代わりに、App Store の設定で「大容量アプリを外付けドライブにダウンロードしてインストール」を有効にすることを推奨します。
:::

## 署名設定

| 設定 | 説明 | デフォルト |
|------|------|----------|
| 自動再署名 | データディレクトリ移行後、関連アプリに対して自動的に Ad-hoc 再署名を実行 | オフ |
| ログイン時自動再署名 | ユーザーがログインするたびに署名が期限切れの移行済みアプリを自動再署名 | オン |

有効にすると、各データディレクトリ移行時に元の署名が自動的にバックアップされ、移行後の「破損」プロンプトを回避するために再署名が実行されます。

ログイン時自動再署名は、LaunchAgent（`com.shimoko.AppPorts.re-sign`）として登録され、ユーザーがログインするたびに署名が期限切れの移行済みアプリを自動的に再署名します。この機能はデフォルトで有効になっており、設定で無効にできます。

::: tip 💡 リンク済みアプリの自動再署名
リンク済みアプリ（ステータス：「Linked」）の場合、自動再署名は Stub Portal シェルやシンボリックリンクの背後にある**外部の実アプリパス**を自動的に解決し、実際のアプリケーションパッケージに署名変更が適用されることを保証します。バックアップと再署名は実アプリの Bundle ID を識別子として使用します。
:::

## ログ設定

| 設定 | 説明 | デフォルト |
|------|------|----------|
| ログを有効にする | 実行時のログをファイルに書き込む | オン |
| 最大ログサイズ | ログファイルがこのサイズを超えた場合、古い半分を自動的に切り詰める | 2 MB |
| ログの場所 | ログファイルの保存パス | `~/Library/Application Support/AppPorts/AppPorts_Log.txt` |

### ログ操作

| 操作 | 説明 |
|------|------|
| Finder で表示 | ログファイルを含むディレクトリを開く |
| 診断パッケージをエクスポート | ログ、操作記録、システム情報を含む ZIP ファイルを生成 |
| ログをクリア | 現在のログファイルの内容をクリア |

ログの詳細な説明については、[ログと診断](/ja/logging)をご参照ください。
