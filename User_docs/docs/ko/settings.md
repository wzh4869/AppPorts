# 설정

AppPorts의 설정 페이지는 메인 창 오른쪽 상단의 톱니바퀴 아이콘을 통해 접근할 수 있습니다.

## App Store 및 iOS 설정

| 설정 | 설명 | 기본값 |
|------|------|--------|
| App Store 앱 마이그레이션 | App Store 앱의 마이그레이션을 허용. macOS 15.1 미만 버전에서는 수동으로 활성화해야 함 | 꺼짐 |
| iOS 앱 마이그레이션 | iOS/iPadOS 앱 (Mac 버전)의 마이그레이션을 허용 | 꺼짐 |

::: tip 💡 macOS 15.1 이상 사용자
macOS 15.1 이상은 App Store 앱의 외장 드라이브 네이티브 설치를 지원합니다. AppPorts의 마이그레이션 토글 대신 App Store 설정에서 "외장 드라이브에 대용량 앱 다운로드 및 설치"를 활성화하는 것을 권장합니다.
:::

## 서명 설정

| 설정 | 설명 | 기본값 |
|------|------|--------|
| 자동 재서명 | 데이터 디렉토리 마이그레이션 후 관련 앱에 대해 자동으로 Ad-hoc 재서명을 실행 | 꺼짐 |
| 로그인 시 자동 재서명 | 사용자가 로그인할 때마다 서명이 만료된 마이그레이션된 앱을 자동으로 재서명 | 켜짐 |

활성화하면 각 데이터 디렉토리 마이그레이션 시 자동으로 원본 서명을 백업하고 재서명을 실행하여 마이그레이션 후 "손상됨" 메시지를 방지합니다.

로그인 시 자동 재서명은 LaunchAgent(`com.shimoko.AppPorts.re-sign`)로 등록되어 사용자가 로그인할 때마다 서명이 만료된 마이그레이션된 앱을 자동으로 재서명합니다. 이 기능은 기본적으로 활성화되어 있으며 설정에서 비활성화할 수 있습니다.

::: tip 💡 연결된 앱의 자동 재서명
연결된 앱 (상태: 'Linked')의 경우, 자동 재서명은 Stub Portal 셸이나 심볼릭 링크 뒤의 **실제 외부 앱 경로**를 자동으로 해결하여 실제 애플리케이션 패키지에 서명 변경이 적용되도록 보장합니다. 백업과 재서명은 실제 앱의 Bundle ID를 식별자로 사용합니다.
:::

## 로깅 설정

| 설정 | 설명 | 기본값 |
|------|------|--------|
| 로깅 활성화 | 런타임 로그를 파일에 기록 | 켜짐 |
| 최대 로그 크기 | 로그 파일이 이 크기를 초과하면 이전 절반을 자동으로 잘라냄 | 2 MB |
| 로그 위치 | 로그 파일 저장 경로 | `~/Library/Application Support/AppPorts/AppPorts_Log.txt` |

### 로그 작업

| 작업 | 설명 |
|------|------|
| Finder에서 보기 | 로그 파일이 포함된 디렉토리를 엽니다 |
| 진단 패키지 내보내기 | 로그, 작업 기록 및 시스템 정보를 포함하는 ZIP 파일 생성 |
| 로그 지우기 | 현재 로그 파일 내용을 지움 |

자세한 로그 설명은 [로깅 및 진단](/ko/logging)을 참조하세요.
